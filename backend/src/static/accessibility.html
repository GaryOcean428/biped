Accessibility Statement

Last Updated: January 4, 2025

Effective Date: January 4, 2025

1. Our Commitment to Accessibility

Braden Pty Ltd is committed to ensuring that the Biped Platform is accessible to everyone, including people with disabilities. We believe that all users should have equal access to our digital marketplace connecting skilled tradespeople with customers, regardless of their abilities or assistive technologies they may use.

2. Accessibility Standards

We strive to conform to the following accessibility standards:

  • Web Content Accessibility Guidelines (WCAG) 2.1 Level AA: International standard for web accessibility
  • Section 508: U.S. federal accessibility standards
  • EN 301 549: European accessibility standard
  • Disability Discrimination Act 1992 (Australia): Australian accessibility requirements

3. Accessibility Features

3.1 Navigation and Structure

  • Consistent navigation structure across all pages
  • Clear headings and page structure using proper HTML semantics
  • Skip links to main content and navigation areas
  • Logical tab order for keyboard navigation
  • Descriptive page titles for each section

3.2 Visual Design

  • High contrast color schemes meeting WCAG AA standards
  • Scalable text that can be enlarged up to 200% without loss of functionality
  • Clear visual focus indicators for interactive elements
  • Consistent visual design patterns throughout the platform
  • Alternative text for all meaningful images

3.3 Keyboard Navigation

  • Full keyboard accessibility for all interactive elements
  • Visible focus indicators on all focusable elements
  • Logical tab order throughout the interface
  • Keyboard shortcuts for common actions
  • No keyboard traps that prevent navigation

3.4 Screen Reader Support

  • Proper ARIA (Accessible Rich Internet Applications) labels and descriptions
  • Semantic HTML markup for optimal screen reader compatibility
  • Descriptive link text that makes sense out of context
  • Form labels and error messages clearly associated with inputs
  • Live regions for dynamic content updates

4. Platform-Specific Accessibility

4.1 Service Matching

  • Service categories organized with clear hierarchy
  • Search functionality with filter options accessible via keyboard
  • Provider profiles with structured information for screen readers
  • Clear pricing and availability information

4.2 Communication Tools

  • Message interfaces compatible with assistive technologies
  • Clear timestamps and sender identification
  • Notification preferences for different communication channels
  • Alternative formats for important communications

4.3 Payment Processing

  • Secure payment forms with clear labels and instructions
  • Error messages that clearly identify issues and solutions
  • Multiple payment options to accommodate different needs
  • Clear confirmation and receipt information

4.4 AI-Powered Features

  • Alternative methods for users who cannot use computer vision features
  • Manual verification options for AI-generated recommendations
  • Clear explanations of how AI features work
  • Opt-out options for users who prefer manual processes

5. Supported Assistive Technologies

Our platform has been tested with and supports the following assistive technologies:

5.1 Screen Readers

  • JAWS (Job Access With Speech)
  • NVDA (NonVisual Desktop Access)
  • VoiceOver (macOS and iOS)
  • TalkBack (Android)
  • Windows Narrator

5.2 Voice Control

  • Dragon NaturallySpeaking
  • Windows Speech Recognition
  • Voice Control (macOS)
  • Voice Access (Android)

5.3 Magnification Software

  • ZoomText
  • MAGic
  • Windows Magnifier
  • macOS Zoom

6. Browser Compatibility

Our platform is compatible with the following browsers when used with assistive technologies:

  • Google Chrome (current and previous major version)
  • Mozilla Firefox (current and previous major version)
  • Safari (current and previous major version)
  • Microsoft Edge (current and previous major version)

7. Mobile Accessibility

Our mobile-responsive design includes accessibility features for mobile devices:

  • Touch targets meet minimum size requirements (44x44 pixels)
  • Adequate spacing between interactive elements
  • Support for mobile screen readers (VoiceOver, TalkBack)
  • Responsive design that maintains functionality at all screen sizes
  • Support for voice control on mobile devices

8. Known Limitations

While we strive for full accessibility, we acknowledge the following current limitations:

  • Some computer vision features may not be fully accessible to users with visual impairments
  • Certain third-party integrations may have accessibility limitations
  • Some older browser versions may not fully support all accessibility features
  • Live chat features may have limited screen reader support during peak usage

9. Ongoing Improvements

We are committed to continuously improving accessibility through:

  • Regular accessibility audits by third-party experts
  • User testing with people with disabilities
  • Staff training on accessibility best practices
  • Monitoring of accessibility standards and guidelines
  • Integration of accessibility considerations into our development process

10. Alternative Formats

We can provide information in alternative formats upon request:

  • Large print versions of important documents
  • Audio recordings of written content
  • Braille versions of key information
  • Plain text versions of complex documents
  • Sign language interpretation for video content

We work with third-party providers for certain platform features. While we cannot guarantee the accessibility of all third-party content, we:

  • Require accessibility statements from major service providers
  • Provide alternative methods when third-party content is not accessible
  • Work with vendors to improve accessibility where possible
